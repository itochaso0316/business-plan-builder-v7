### 接続制約（重要）

`.claude/settings.json` は以下の設定になっている：

```json
{
  "mcpServers": {
    "hojokin": {
      "command": "hojokin-mcp",
      "args": [],
      "env": { "SUPABASE_URL": "...", "SUPABASE_ANON_KEY": "..." }
    }
  }
}
```

`"command": "hojokin-mcp"` はシステム PATH 上に `hojokin-mcp` バイナリがあることを前提とするが、現在このコマンドは `~/Projects/hojokin-jp/mcp-server/.venv/bin/` にのみ存在し、システム PATH には含まれない。

本テストでは `.venv/bin/hojokin-mcp` を直接起動して動作確認を行い、stdio MCP 通信レイヤーの初期化が正常に完了することを確認済み。`settings.json` の `command` フィールドをフルパスに変更すれば Claude Code から直接呼び出し可能。

### システム Python の制約（参考）

システム Python（3.9.6）では `pip3 install hojokin-mcp` が `mcp>=1.0.0` の解決に失敗する（PyPI の `mcp` パッケージが Python 3.9 非対応）。本テストは `.venv`（Python 3.13）で実施した。

### 課題（優先度順）

1. **[P0] settings.json の `command` 設定不備（Claude Code から直接呼び出しが不可）**  
   `"command": "hojokin-mcp"` がシステム PATH に存在しないため、BPB v7 の Claude Code セッションから `hojokin_match` ツールが呼び出せない。  
   → `settings.json` の `command` をフルパス（`.venv/bin/hojokin-mcp`）または `python3 server.py` に変更する。

2. **[P0] システム Python（3.9）での hojokin-mcp インストール不可**  
   Claude Code の実行環境（`/usr/bin/python3 = 3.9.6`）では `mcp>=1.0.0` が PyPI から解決できない。  
   → `.venv`（Python 3.13）の利用を前提とした `settings.json` 設定への切り替え、または Homebrew で Python 3.10+ をシステムに追加。

3. **[P1] deadline（締切日）の収集率向上（2.1%）**  
   全1,000件中 21件しか締切日が設定されていない。BPB Phase 4.3 で「申請期限の確認」機能が実質機能しない。  
   → hojokin-jp クローラーの締切日パース処理を強化。公式ページからの構造化抽出率改善。

4. **[P2] Python 3.13 での `TimeoutError` 例外型の変更に対応**  
   Python 3.13 では `urllib.error.URLError` の代わりに `TimeoutError` が raise される。`server.py` は `except Exception` で対応済みだが、明示的な型 catch があれば修正が必要。  
   → `server.py` のエラーハンドリングに `TimeoutError` を明示的に追加。

5. **[P2] 全国補助金の重複排除**  
   「デジタル化・AI導入補助金（通常枠）」と「デジタル化・AI導入補助金2026（通常枠）」が別レコードとして存在し、同一クエリでいずれもヒットする。  
   → 年度更新に伴う旧レコードのステータス管理（`status: inactive` 設定等）を整備。
