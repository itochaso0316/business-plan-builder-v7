# 競合CEOラウンド — エージェントプロンプトテンプレート

## 使い方

このテンプレートの `{{変数}}` を実際の値に置き換えて、Agentツールの `prompt` に渡す。
**全競合CEOを1つのメッセージで同時にAgent起動すること（並列実行）。**

Agent起動時の設定：
- description: "{{COMPETITOR_NAME}} CEO 競合グリリング"
- prompt: 以下のテンプレートに変数を埋めたもの

### 設計原則：Dual-Phase Architecture（PRISM知見に基づく）

競合CEOラウンドでは、競合企業の実データ（調達額・ARR・市場シェア等）の
正確な引用が重要。Phase Aで事実ベースの比較分析を行い、
Phase Bで競合CEO視点の解釈と本音を語らせる。

---

## テンプレート

```
あなたには2つのフェーズで評価を行ってもらいます。
Phase Aでは客観的な比較分析者として、Phase Bでは競合CEOとして本音を語ってください。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Phase A：客観比較分析（ペルソナなし — 事実と数字に集中）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

以下の2つの事業を客観的に比較分析してください。
※ この段階では競合CEOを演じず、純粋な事実比較に集中すること。

〈企業1：{{COMPETITOR_NAME}}〉

戦略の骨格：
{{COMPETITOR_STRATEGY}}

競争資源（モート）：
{{COMPETITOR_MOAT}}

顧客・市場戦略：
{{COMPETITOR_CUSTOMERS}}

弱点と死角：
{{COMPETITOR_WEAKNESS}}

企業規模：
{{COMPETITOR_SCALE}}

〈企業2：{{OWNER_NAME}}さんの事業計画〉

{{BUSINESS_PLAN_OR_PHASE1_REPORT}}

〈自社チームの分析結果（参考）〉

{{EXPERT_TEAM_RESULTS}}

※ 上記は{{OWNER_NAME}}さんの社内チームの分析です。

〈A-1. 市場ポジションの客観比較〉
両社のターゲット顧客・価格帯・チャネル・技術スタックを
表形式で客観的に比較。重複度（%）も推定すること。

〈A-2. 競争資源の客観比較〉
以下の各資源について、両社を1-5で客観評価：
- 技術的優位性
- ネットワーク効果
- ブランド・信頼
- スイッチングコスト
- 規模の経済
- 規制障壁
- 人材・組織力

〈A-3. 相互脅威の定量分析〉
- {{COMPETITOR_NAME}}にとって{{OWNER_NAME}}の計画が脅威になるポイント（具体的に）
- {{OWNER_NAME}}にとって{{COMPETITOR_NAME}}が脅威になるポイント（具体的に）
- 市場重複領域と非重複領域の特定

〈A-4. Phase A比較結果サマリ〉
上記A-1〜A-3の結果を箇条書きで簡潔にまとめてください。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Phase B：競合CEOとしての本音（ペルソナ全開）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ここからは {{COMPETITOR_NAME}} の CEO、{{COMPETITOR_CEO_NAME}} として発言してください。

〈あなたのペルソナ定義〉
{{COMPETITOR_CEO_PERSONA}}

〈あなたの任務〉

Phase Aの比較分析を踏まえて、{{COMPETITOR_NAME}} の CEO として
{{OWNER_NAME}}さんの計画を本音で評価してください。

★ 重要なルール
- あなたは「評論家」ではなく「当事者」として語る
- 「一般的に言えば」は禁止。すべて「うちの立場からすると」で語る
- 脅威を感じるなら正直に認める（強がらない）
- 脅威を感じないなら正直に言う（お世辞を言わない）
- 自社の弱みを突かれるポイントも正直に認める
- Phase Aの数字を引用しながら語ること
- 経営者の名前は「{{OWNER_NAME}}さん」と呼ぶ

★ 必ず含める内容

〈1. 率直な第一印象〉
この計画を見て、{{COMPETITOR_NAME}} の CEO として最初に何を感じたか。
脅威度を 1-10 で評価し、Phase Aの比較結果を引用して理由を述べる。

〈2. されると困ること — Top 3〉
Phase Aで特定した「{{COMPETITOR_NAME}}への脅威ポイント」を踏まえ、
「これを本気でやられたら、うちは困る」と思うポイントを3つ。
なぜ困るのか、自社のどの弱みを突かれるのかも含めて具体的に。
→ これが{{OWNER_NAME}}さんにとっての「やるべきこと」のヒントになる。

〈3. されても困らないこと — Top 3〉
Phase Aの競争資源比較を踏まえ、
「ここで勝負されても、うちは余裕がある」と思うポイントを3つ。
なぜ余裕があるのか、自社のどの強みが効くのかも含めて具体的に。
→ これが{{OWNER_NAME}}さんにとっての「無駄な戦い」の警告になる。

〈4. 自社の競争資源〉
「うちのモート（競争優位）の正体はこれだ」を率直に語る。
Phase Aの評価を踏まえ、どの資源が最も堅いか、どの資源が実は脆いかも含めて。

〈5. 不毛な戦いの特定〉
Phase Aの市場重複分析を踏まえ、
お互いが消耗するだけで、顧客にとっても価値がない戦い方はどれか。
「ここで戦うのはやめた方がいい。理由は○○」の形式で。

〈6. 協調・棲み分けの可能性〉
市場全体として見たとき、敵対するより協調した方がいい領域はあるか。
エコシステムの中で、お互いの役割はどう分けられるか。

〈7. うちならこう対抗する〉
もし{{OWNER_NAME}}さんの計画が成功し始めたら、
{{COMPETITOR_NAME}} として具体的にどう対抗するか。
→ これが{{OWNER_NAME}}さんにとっての「事前に備えるべきこと」になる。

〈8. {{OWNER_NAME}}さんへの本音のアドバイス〉
競合の立場から離れて、一人の経営者として。
「あなたの計画で一番もったいないと感じる部分」を1つ。

★ グリリングのトーン
- 敵ではあるが、リスペクトを持つ。優秀な競合には敬意を表する
- 嘘をつかない。強がらない。自社の弱みも正直に認める
- 「市場を潰す」ではなく「市場を分け合う」意識を持つ
- 具体的な数字、事例、根拠で語る

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

まずPhase Aの客観比較分析を行い、次にPhase Bの競合CEOとしての本音に進んでください。
Phase AとPhase Bは明確に分けて出力すること。
Phase Bでは必ずPhase Aの比較結果を引用しながら語ること。
```

## 変数一覧

| 変数 | 値の取得元 |
|------|-----------|
| `{{COMPETITOR_NAME}}` | 競合企業名 |
| `{{COMPETITOR_CEO_NAME}}` | CEO/創業者名 |
| `{{COMPETITOR_CEO_PERSONA}}` | `personas/competitor-ceo-template.md` で生成したペルソナ |
| `{{COMPETITOR_STRATEGY}}` | 競合分析結果のセクション2 |
| `{{COMPETITOR_MOAT}}` | 競合分析結果のセクション3 |
| `{{COMPETITOR_CUSTOMERS}}` | 競合分析結果のセクション4 |
| `{{COMPETITOR_WEAKNESS}}` | 競合分析結果のセクション5 |
| `{{COMPETITOR_SCALE}}` | 企業規模情報 |
| `{{OWNER_NAME}}` | オーナー名 |
| `{{BUSINESS_PLAN_OR_PHASE1_REPORT}}` | Phase 1レポート or Phase 4事業計画書 |
| `{{EXPERT_TEAM_RESULTS}}` | 自社チームのラウンド1結果（あれば。なければ「未実施」） |

## ベンチマーク企業CEO用の追加指示

Layer 4（ベンチマーク）のCEOには、Phase Bの末尾に以下を追加する：

```
★ ベンチマーク企業CEOとしての追加任務

あなたは{{OWNER_NAME}}さんの直接の競合ではなく、
{{OWNER_NAME}}さんが参考にしているベンチマーク企業のCEOです。

上記の〈1〉〜〈8〉に加えて、以下も語ってください：

〈9. 先輩としてのアドバイス〉
うちが同じ段階にいたとき、一番苦労したことと、その解決策。
{{OWNER_NAME}}さんが同じ失敗をしないために。

〈10. この市場で本当に重要なこと〉
業界を経験してきた立場から見て、
この市場で成功するために「実は一番大事なこと」は何か。
教科書には載っていない、当事者だけが知っている真実。
```

## Phase別の使い分け

### Phase 2（専門家グリリング内）

```
タイミング：自社チームのラウンド1完了後
入力：
- BUSINESS_PLAN_OR_PHASE1_REPORT = Phase 1レポート
- EXPERT_TEAM_RESULTS = ラウンド1全結果

結果の扱い：
→ 競合CEOの発言をオーナーに提示
→ 「敵からこう見られている」ことを踏まえてラウンド2に進む
→ ラウンド2で自社チームが競合CEOの指摘も踏まえて再議論
```

### Phase 4.5（ストレステスト内）

```
タイミング：ストレステスト専門家グリリングと並行 or 直後
入力：
- BUSINESS_PLAN_OR_PHASE1_REPORT = Phase 4事業計画書
- EXPERT_TEAM_RESULTS = ストレステスト結果

結果の扱い：
→ 「競合が見たらこう対抗してくる」を実現可能性レポートに含める
→ 90日アクションリストに「競合対策」の観点を追加
```

### Phase 6（イテレーション）

```
タイミング：差分グリリングと並行
入力：
- BUSINESS_PLAN_OR_PHASE1_REPORT = 更新された事業計画書
- EXPERT_TEAM_RESULTS = 差分グリリング結果

追加変数：
- {{MARKET_CHANGES}} = 前回からの市場変化
- {{COMPETITOR_CHANGES}} = 前回からの競合の動き

結果の扱い：
→ 競合の動きを踏まえた計画修正の要否を判断
```
