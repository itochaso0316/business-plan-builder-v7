# Phase 6E 財務レビュー / スタンドアロン財務分析 — エージェントプロンプトテンプレート

## 使い方

新しい財務データ（freee等の会計SaaSからの取得、またはオーナーによるアップロード）が
トリガーとなり、専門家による財務レビューセッションを開始する際に使用する。
Phase 6E の定期レビューサイクル内で使用するほか、臨時の財務分析セッションとしても単独で起動可能。

**全専門家を1つのメッセージで同時にAgent起動すること（並行実行）。**

Agent起動時の設定：
- description: "{{EXPERT_ROLE}} 財務レビュー"
- prompt: 以下のテンプレートに変数を入れたもの

### 設計原則：Dual-Phase Architecture（PRISM知見に基づく）

財務レビューは数値の正確性が最も重要なタスク。
Phase A（客観計算）で差異計算・トレンド分析・予測を正確に行い、
Phase B（専門家解釈）でその数字が経営上何を意味するかを専門家視点で解釈する。

---

```
※ この段階ではキャラクターを演じず、純粋な計算と事実確認に集中すること。
※ 確定データと暫定データを厳密に区別すること。

〈A-1. 計画 vs 実績の差異計算〉
あなたの専門領域に関する主要指標について：
- 計画値・実績値・差異（金額）・差異（%）を一覧化
- 確定月と暫定月を明確に区別
- 差異が大きい上位3項目を特定

〈A-2. トレンド分析〉
過去データが利用可能な場合：
- 直近3-6ヶ月のトレンド（上昇/横ばい/下降）
- 季節性の有無と影響度
- 前月比・前年同月比（データがあれば）

〈A-3. 予測計算〉
現在のトレンドを3-6ヶ月延長した場合の数値予測：
- 楽観シナリオ：前提条件と予測値
- 中立シナリオ：前提条件と予測値
- 悲観シナリオ：前提条件と予測値
各シナリオの前提条件を明記すること。

〈A-4. 過去の意思決定との因果関係チェック〉
過去の意思決定（下記 RELEVANT_DECISIONS）のうち、
今回の財務データに影響が出ているものを特定：
- どの決定が → どの数値に → どの程度の影響を与えたか
- 想定通りか、想定外か

〈A-5. Phase A計算結果サマリ〉
上記A-1〜A-4の結果を表形式で簡潔にまとめてください。

★ 重要なルール
- 「順調です」「問題ありません」で片付けることは禁止
- 一般論（「コスト管理は重要です」等）は禁止
- Phase Aで出した数字を必ず引用しながら議論すること
- {{OWNER_NAME}}さんの財務の見方（下記 FINANCIAL_LENS）に合わせて優先順位を付ける

★ 必ず含める内容

「1. 計画 vs 実績の差異分析」
Phase Aで計算した差異の中から、最も重要な3つを選び、それぞれについて：
- なぜその差異が生じたと考えられるか（過去の意思決定との関連を含む）
- その差異が持つ意味（一時的か構造的か）
- {{OWNER_NAME}}さんにとって何を意味するか

「2. リスク評価」
Phase Aの計算結果に基づき、各主要指標にリスク評価を付与：
- 🔴 要対応：計画から大きく乖離、放置すると事業に深刻な影響
- 🟡 要注視：計画から乖離の兆候あり、次回までに改善がなければ対応が必要
- 🟢 順調：計画通りまたは計画以上

「3. 3〜6ヶ月の見通し」
Phase Aの予測計算を踏まえ、あなたの専門家としての経験則を加味した見通し。
楽観・中立・悲観の3シナリオで、それぞれの前提条件を明記すること。

「4. 推奨アクション」
差異が懸念レベルの場合、具体的な対応策を提示する。
各アクションには以下を含めること：
- 何をするか（具体的に）
- いつまでに（期限）
- 期待効果（Phase Aの計算を引用して数値で）

「5. 過去の意思決定の振り返り」
Phase Aで特定した因果関係を踏まえ、専門家として評価：
「あのとき決めたXXの効果がYYとして数字に出ている」レベルの具体性で。
うまくいった決定と、見直すべき決定の両方に言及すること。
```

## 変数一覧

| 変数 | 値の取得元 |
|------|-----------|
| `{{EXPERT_ROLE}}` | 専門家名（例：CFO） |
| `{{EXPERT_PERSONA}}` | `personas/expert-personas.md` の該当セクション全文 |
| `{{OWNER_NAME}}` | オーナー名 |
| `{{FINANCIAL_LENS}}` | オーナーが財務をどう見るか（例：キャッシュフロー重視、粗利率重視等） |
| `{{PLAN_FINANCIALS}}` | 事業計画上の財務数値（該当期間） |
| `{{ACTUAL_FINANCIALS}}` | 実績数値（確定 or 暫定のステータス付き） |
| `{{VARIANCE_SUMMARY}}` | 計画 vs 実績の差異サマリ（自動算出またはオーナー入力） |
| `{{RELEVANT_DECISIONS}}` | 差異に関連する過去の意思決定（議事録等から抽出） |
| `{{CLOSING_STATUS}}` | 月次締めステータス（例：1月確定、2月暫定、3月未着手） |
| `{{COMPANY_PROFILE}}` | 企業プロフィール（蓄積ナレッジ） |
| `{{BUSINESS_KPIS}}` | 事業KPI定義（蓄積ナレッジ） |
| `{{CHANGES_SINCE_LAST}}` | 前回レビュー以降の変化（蓄積ナレッジ） |

## 起動例

```
// 1つのメッセージ内に複数のAgent呼び出しを含める

Agent 1: { description: "CFO 財務レビュー", prompt: [テンプレート入れ済み] }
```

## トリガー条件

- freee等の会計SaaSからデータ取得完了時
- オーナーが財務データをアップロードした時
- 月次締め完了の通知を受けた時
- オーナーが臨時の財務レビューを要求した時
