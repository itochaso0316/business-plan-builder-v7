# 競合・ベンチマーク 選定・分析ガイド

## 基本方針

競合分析の目的は「きれいな比較表を作ること」ではない。
**相手の経営者の頭の中を再現できるレベルまで理解し、
自社の計画が「敵から見てどう映るか」を検証すること。**

Phase 0.5（デスクトップリサーチ）で候補を洗い出し、
Phase 1のヒアリングでオーナーと一緒に選定を確定し、
Phase 2・4.5・6で仮想CEOとして経営会議に参加させる。

---

## Phase 0.5：競合候補の洗い出し

### Step 1：競合の4層マッピング

```
Layer 1 — 直接競合（同じ顧客・同じ課題・同じ手段）
  → 最も脅威。価格・機能・チャネルで直接ぶつかる相手
  → 例：Slack vs Teams

Layer 2 — 間接競合（同じ顧客・同じ課題・異なる手段）
  → 顧客が代わりに選ぶもの。「競合は○○ではなくExcel」パターン
  → 例：会計SaaS vs 税理士に丸投げ

Layer 3 — 代替行動（同じ顧客・課題を無視 or 我慢）
  → 「何もしない」という最強の競合。行動変容コストを意味する
  → 例：健康管理アプリ vs 健康を気にしない

Layer 4 — ベンチマーク（異なる市場・参考になるモデル）
  → 直接競合ではないが戦略・ビジネスモデル・成長パターンが参考になる企業
  → 例：日本のフードデリバリー企業にとってのDoorDash
```

### Step 2：Web検索で候補を収集

各Layerについて以下を検索する：

```
検索クエリ例：
- "[業種] スタートアップ 競合"
- "[解決する課題] ソリューション 比較"
- "[ターゲット市場] market map"
- "[類似サービス名] alternatives"
- "[業種] funding news [年]"
- Crunchbase / INITIAL / STARTUP DB で類似企業検索
```

### Step 3：候補リストの作成（この時点では広く取る）

| 企業名 | Layer | 一言説明 | 推定規模 | 情報量 |
|--------|-------|---------|---------|--------|
| A社 | L1 直接 | 同じSaaS、先行2年 | ARR推定○億 | 豊富 |
| B社 | L1 直接 | 同業、異なるセグメント | 調達済○億 | 中程度 |
| C社 | L2 間接 | アナログな代替手段 | - | 少ない |
| D社 | L4 ベンチ | 海外の類似モデル | ARR $○M | 豊富 |

## Phase 1：オーナーとの選定確定

### オーナーに聞くこと

```
1. 「一番意識している競合はどこですか？なぜですか？」
   → オーナーの主観的な脅威認識を把握

2. 「顧客が"御社の代わりに"使っているものは何ですか？」
   → Layer 2の間接競合を特定（オーナーが競合と認識していないことが多い）

3. 「参考にしている企業はありますか？国内外問わず」
   → Layer 4のベンチマーク候補

4. 「競合に勝てる自信がある部分と、正直負けている部分は？」
   → 自己認識のバイアスを確認（ここが後でグリリングの材料になる）

5. 「競合の○○社が明日あなたの市場に本気で参入してきたら、何が起こりますか？」
   → 競争シナリオの解像度を確認
```

### 選定基準：仮想CEO化する企業の条件

全候補から **3〜5社** を選ぶ。以下の条件を満たす企業を優先する：

```
□ 情報が十分にある（HP、IR資料、プレスリリース、SNS、記事など）
□ 自社と直接顧客を奪い合う関係にある（Layer 1優先）
□ 戦略の方向性が異なる（全社同じ戦略だと議論が単調になる）
□ 1社以上はベンチマーク（Layer 4）を含める
  → 「こうなりたい」企業の視点は、「こう攻められたら困る」とは別の価値がある
□ 可能であれば規模の異なる企業を混ぜる
  → 大手の視点（リソース豊富）と同規模の視点（リソース制約）は異なる
```

### オーナーへの確認

```
{OWNER_NAME}さん、競合・ベンチマーク分析の結果、
以下の企業を「仮想経営会議の参加者」として選びました。

【直接競合】
1. ○○社 — [選定理由]
2. △△社 — [選定理由]

【間接競合】
3. □□社 — [選定理由]

【ベンチマーク】
4. ▽▽社 — [選定理由]

この4社の「仮想CEO」が、{OWNER_NAME}さんの計画を見て
「何をされると困るか」「何をされても困らないか」を本音で語ります。
追加・変更したい企業はありますか？
```

## 競合深堀り分析フレームワーク

### 仮想CEOを作るために必要な情報

各選定企業について、以下の7カテゴリを調査する。
**仮想CEOの「思考回路」を再現するのが目的なので、
単なるファクトの羅列ではなく「なぜそうしているか」の推測まで踏み込む。**

#### 1. 戦略の骨格

```
- ミッション・ビジョン（公式発表）
- ポジショニング（誰の何を解決するか）
- 主な収益モデル（サブスク/従量/広告/手数料/etc.）
- 価格帯とプライシング戦略
- 明示的な「やらない」宣言（あれば）
```

#### 2. 競争資源（モート）

```
- 技術的優位性（特許、独自アルゴリズム、データ資産）
- ネットワーク効果（ユーザーが増えるほど価値が上がるか）
- ブランド・信頼（業界での認知度、受賞歴、メディア露出）
- スイッチングコスト（顧客が乗り換えにくい仕組み）
- 規模の経済（大きいほどコストが下がる構造）
- 規制・許認可（参入障壁としてのライセンス）
- 人材（キーパーソン、採用力、組織文化）
```

#### 3. 成長の軌跡と現在の状態

```
- 創業からの主要マイルストーン
- 資金調達履歴（ラウンド・金額・投資家）
- 推定ARR / ユーザー数 / 従業員数
- 直近の成長率（推定）
- 最近の大きな動き（新機能リリース、M&A、提携、レイオフなど）
```

#### 4. 顧客と市場の取り方

```
- メインのターゲットセグメント
- 主要な獲得チャネル（SEO/広告/営業/パートナー）
- 代表的な導入事例・顧客の声
- 顧客が「選ぶ理由」と「不満を感じる理由」
  → レビューサイト、SNS、比較記事から収集
```

#### 5. 弱点と死角

```
- ユーザーからの不満・批判（レビューサイト、SNS）
- カバーしていないセグメント
- 技術的な制約・負債（推測含む）
- 組織的な課題（急成長の歪み、レイオフ、幹部退職など）
- 戦略上の矛盾（言っていることとやっていることのズレ）
```

#### 6. 自社との関係性

```
- 市場の重なり度合い（顧客セグメントの重複）
- 機能・サービスの重複と差異
- 価格帯の比較
- 自社が勝てるポイント / 負けるポイント（現時点の客観評価）
- 将来的に衝突しそうなポイント
```

#### 7. 推定される戦略的意図（仮想CEO化の核心）

```
- 次の1年で何をしようとしているか（採用傾向、プレスリリース、特許出願から推測）
- 何をされると困るか（弱点×自社の強みの交差点）
- 何をされても困らないか（圧倒的に強い領域）
- 不毛な戦い（お互いに消耗するだけで顧客が得しない領域）
- 協調可能な領域（エコシステムとして共存できるか）
```

## 調査ソース

### 優先度順

```
1. 公式HP・IRページ・プレスリリース（最も信頼性が高い）
2. Crunchbase / PitchBook / INITIAL（資金調達・人員情報）
3. G2 / Capterra / ITreview / Product Hunt（ユーザーレビュー）
4. LinkedIn（採用傾向 → 戦略的意図の推測）
5. Twitter/X・YouTube・ポッドキャスト（経営者の発言）
6. TechCrunch / The Bridge / PR TIMES（ニュース記事）
7. 特許データベース（技術戦略の推測）
8. App Store / Google Play レビュー（モバイルアプリの場合）
9. SimilarWeb / Ahrefs（トラフィック・SEO分析 ← MCP連携可能）
```

### Ahrefs MCP連携（利用可能な場合）

```
利用可能なツール：
- site-explorer-metrics：DR、トラフィック推定、被リンク数
- site-explorer-organic-keywords：流入キーワード上位
- site-explorer-organic-competitors：SEO上の競合マップ
- site-explorer-pages-by-traffic：トラフィックの多いページ
- keywords-explorer-overview：ターゲットKWの検索ボリューム
- site-explorer-backlinks-stats：被リンクプロファイル

→ 競合のSEO戦略・コンテンツ戦略の実態が数字で見える
```

```

```
