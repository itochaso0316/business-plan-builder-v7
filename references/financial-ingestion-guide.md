# 財務データ取込ガイド

## 1. 対応フォーマット

| フォーマット | 読込方法 | 備考 |
|---|---|---|
| `.xlsx` / `.xlsm` | xlsx スキル | 推奨。構造化データとして最も扱いやすい |
| `.pdf` | pdf スキル | OCR 経由。表の読み取り精度に注意 |
| `.csv` / `.tsv` | 直接読み込み | エンコーディング（UTF-8 / Shift_JIS）を確認 |

---

## 2. 抽出プロトコル

財務データを受け取った場合、以下を抽出する。

### 抽出対象

- **主要勘定科目**: 売上高、売上原価、粗利、販管費、営業利益、経常利益、当期純利益
- **主要比率**: 粗利率、営業利益率、経常利益率
- **期間変化**: 前期比、前年同期比
- **BS項目**（提供された場合）: 総資産、純資産、自己資本比率、流動比率
- **CF項目**（提供された場合）: 営業CF、投資CF、財務CF

### 保存ルール

- 生データ（元ファイル）はナレッジベースに格納しない
- 抽出した数値とインサイトを **Markdown 形式** で保存する
- 数値には必ず出典期間と確定/暫定の区分を付記する

## 3. ファイル命名規則

```
financials/YYYY-QN-{type}.md
```

| 要素 | 説明 | 例 |
|---|---|---|
| `YYYY` | 会計年度 | `2025` |
| `QN` | 四半期 | `q1`, `q2`, `q3`, `q4`, `fy`（通期） |
| `{type}` | 種別 | `pl`（損益）, `bs`（貸借）, `cf`（CF）, `summary`（総合） |

例:

```
financials/2025-q4-pl.md
financials/2025-fy-summary.md
financials/2026-q1-pl.md
```

## 4. Financial Lens ダイアログ

初回の財務データアップロード時、オーナーに以下の6つの質問を行い、`financial-lens.md` に記録する。

### 質問項目

1. **最初に見る数字は何ですか？**
   - 例: 「まず粗利率を見る」「売上の前月比を最初にチェック」

2. **"良い月" と "悪い月" の判断基準は何ですか？**
   - 例: 「営業利益が黒字なら良い月」「粗利率40%以上なら良い」

3. **季節性はありますか？特に売上が集中・低迷する時期は？**
   - 例: 「12月が最盛期、2月が閑散期」

4. **現在の経営で最も気になっている数字は何ですか？**
   - 例: 「人件費率が上がっている」「広告費のROIが見えない」

5. **数字を見るとき、誰かと比較していますか？（競合・業界平均・過去の自社）**
   - 例: 「業界平均の営業利益率と比べている」

6. **この事業の財務的なゴールを一言で言うと？**
   - 例: 「来期までに営業利益率10%」「3年で売上2倍」

これらの回答は `financial-lens.md` に保存し、以降の財務分析・事業計画作成時の判断基準として参照する。

以下のサービスとMCP連携が可能。

| サービス | 取得可能データ | 備考 |
|---|---|---|
| freee | 仕訳、試算表、PL/BS | API連携設定が必要 |
| Money Forward | 仕訳、試算表、PL/BS | API連携設定が必要 |

## 6. 締め作業ステータス管理

### 必須確認事項

財務データを受け取ったら、**必ず**以下を確認する:

> **「何月分まで締め作業は完了していますか？」**

### ステータス区分

| ステータス | 意味 | 事業計画での扱い |
|---|---|---|
| **確定** | 締め作業完了済み。税理士確認済みまたは自社で確定処理済み | 事業計画に「事実」として記載可能 |
| **暫定** | 入力済みだが締め未完了。修正の可能性あり | 予測の参考値としてのみ使用。事業計画本文には使わない |

### `closing-status.md` の記録フォーマット

```markdown
# 締め作業ステータス

## 2025年度
| 月 | ステータス | 確認日 | 備考 |
|---|---|---|---|
| 4月 | 確定 | 2025-06-15 | 税理士確認済 |
| 5月 | 確定 | 2025-07-10 | |
| 6月 | 確定 | 2025-08-12 | |
| 7月 | 暫定 | 2025-09-01 | freee入力済、未確定 |
| 8月 | 未入力 | - | |
```

## 7. 予測値の生成ルール

### 入力データ

- 確定月のトレンド（主要入力）
- 季節性パターン（Financial Lens の質問3から取得）
- 暫定月のデータ（補助的入力として使用）

### 出力フォーマット

予測は必ず **3シナリオのレンジ** で提示する。

| シナリオ | 説明 |
|---|---|
| **楽観（Optimistic）** | 好条件が重なった場合 |
| **基本（Base）** | 最も蓋然性が高いシナリオ |
| **悲観（Pessimistic）** | リスクが顕在化した場合 |

### 事業計画での表記ルール

事業計画書において、**「確定事実」と「予測」を明確に分離** する。

- 確定事実: 「2025年4月-6月実績に基づく」等の出典を明記
- 予測値: 「基本シナリオに基づく予測値」と明記し、前提条件を併記
- 暫定データに基づく記述は「暫定値に基づく参考数値」と注記

## 8. 実績 vs 計画の比較（Actual vs Plan）

### 比較のトリガー

以下の2つが揃った場合に比較分析を実施する:

1. プロジェクトに `financial-model.xlsx`（または同等の計画数値）が存在する
2. 新しい実績データ（確定分）がアップロードされた

### 比較の視点

オーナーの Financial Lens を通して比較する。オーナーが「最初に見る数字」から分析を始め、オーナーの判断基準に沿ってコメントする。

### 乖離の閾値

| 乖離幅 | アクション |
|---|---|
| +/-10% 未満 | 正常範囲。簡潔にコメント |
| +/-10% 以上 20% 未満 | 注意喚起。要因の仮説を提示 |
| **+/-20% 以上** | **Phase 6E トリガー**: 計画見直しレビューを提案 |

Phase 6E トリガーが発動した場合:

1. 乖離の事実を明示する
2. 考えられる要因を3つ程度列挙する
3. 計画の前提条件のどこが変わったかを特定する
4. オーナーに計画修正の要否を確認する
