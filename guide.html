2. システム概要

Claude Code上で動作するSkillで、8つのフェーズを通じて経営者との対話から事業計画を作成します。

Phase 0資料読込
→
Phase 0.5デスクリサーチ
→
Phase 1深層ヒアリング
→
Phase 1.5専門家編成
→
Phase 2専門家会議
→
Phase 3本格リサーチ
→
Phase 4共同執筆
→
Phase 5成果物生成

主な特徴

  • 100+ 専門家を業種に応じて動的に招集
  • 補佐CEO・通訳・議事録係がディスカッションを常時サポート
  • Shopify / GA4 等のMCP連携で既存顧客データを直接分析
  • AI臭くならないデザインでPPT / Word / Excel を生成
  • 経営者が腑に落ちるまで何度でも繰り返すループ設計

3. セットアップ手順

3-1. Claude Code のインストール

# Node.js 18以上が必要

5. 運用ノウハウ & TIPS

Phase 1（ヒアリング）を充実させる

  • Phase 1 が事業計画の質を決める最重要フェーズ。焦らず丁寧に答えるのがコツ
  • 10カテゴリの質問が用意されているが、一度に聞かれるのは最大2つまで
  • 「わからない」と言ってOK — 専門家が Phase 2 で補完してくれる
  • 数字（売上目標、予算等）は概算でもいいので伝える方が精度が上がる

Phase 2（専門家会議）を活用する

  • 専門家チームは業種・目的に応じて 自動で最適編成 される
  • フィードバックは 🔴危険 / 🟡要注意 / 🟢良好 の3段階で提示
  • 「この専門家の意見をもっと深く聞きたい」と指定できる
  • 専門用語は「通訳」が平易に言い換えてくれる
  • 納得できなければ 何度でもやり直せる（納得ループ）

出力品質を上げるコツ

  • PPTテンプレートがあるなら templates/ に置く — デザインの統一感が段違い
  • Web Search MCP を接続しておくと、市場規模・競合データが出典付きで入る
  • Phase 4 では A/B/C案が提示される — 「Aの構成 + Bのトーン」のような組み合わせ指定が有効
  • 「AI臭い」と感じたらその場で指摘すれば即座に調整される

効率的な進め方

  • 既存の事業計画書・リサーチ資料があれば Phase 0 で全部読み込ませる — 重複ヒアリングが激減
  • Shopify / GA4 を接続済みなら、顧客データ分析を Phase 1 で自動実行してくれる
  • 途中で中断しても output/minutes/ に議事録が残るので、再開時に「前回の続きから」と言えばOK
  • 複数事業を比較したい場合は、別ディレクトリで並行して走らせる

よくあるトラブルと対処

症状対処法
Skill が起動しない/skills list で確認 → 再インストール
Web検索が動かないMCP設定ファイル（~/.claude/mcp.json）を確認
PPT生成でエラー/plugin install pptx@skills を再実行
議事録が保存されないmkdir -p output/minutes を実行
Phase が飛んだ「Phase 1 に戻りたい」と明示的に伝える

7. 応用 TIPS

Claude Code の --dangerously-skip-permissions フラグ

ファイル書き込みやコマンド実行の確認をスキップして、完全自動モードで動かせます。信頼できる環境でのみ使用してください。

claude --dangerously-skip-permissions

AI活用仕様書の自動出力

Phase 1 で「AI活用を前提とした事業」と伝えると、Phase 5 で Claude Code 向けのAI活用仕様書（Markdown）が追加出力されます。これをそのまま別の Claude Code セッションに渡せば、実装の起点になります。

締め確認フロー

会計データを使う際は、必ず「何月分まで締め作業は完了していますか？」と確認します。

  • 確定月 → fact（事業計画の根拠として使用）
  • 未締め月 → provisional（予測の参考。fact扱いしない）

この区分は closing-status.md で管理され、データ分析時に常に表示されます。

Financial Lens（財務の読み方学習）

初回の財務データアップロード時に、6つの質問であなた固有の「数字の見方」をヒアリングします：

  1. 月次の数字を見るとき、最初にどこを見ますか？
  2. 一番気になる数字は？
  3. キャッシュフローと利益、どちらを重視しますか？
  4. 月次で何%変動したら危険信号ですか？
  5. 業界のベンチマークと比較することはありますか？
  6. 銀行や投資家によく聞かれる数字は？

回答は financial-lens.md に記録され、以降すべての財務分析がこの視点で行われます。

予測生成

  • 確定月のトレンド + 季節性から向こう3-6ヶ月を推計
  • 楽観 / 基準 / 悲観 のレンジで提示
  • 事業計画書では「確定fact」と「予測」を明確に分離
